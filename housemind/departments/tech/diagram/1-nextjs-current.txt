│                      API ROUTES                                 │

│  POST /api/auth/login          Email/password → JWT cookie      │
│                                                                 │
│  GET  /api/products            List (optional ?category=)       │
│  POST /api/products/[id]/request  Increment availability count  │
│                                                                 │
│  GET  /api/projects            List user's projects             │
│  POST /api/projects            Create project                   │
│  GET  /api/projects/[id]       Detail + products + comments     │
│                                                                 │
│  POST /api/comments            Add comment (membership check)   │
│  POST /api/feedback            Anonymous feedback               │

              ▼
┌─────────────────────────────────────────────────────────────────┐
│                      DATABASE (PostgreSQL)                      │
│                                                                 │
│  users              id, name, email, password_hash, role        │
│  invite_tokens      id, token, email, role, used, created_by    │
│  products           id, name, category, image_url, description, │
│                     status (reference/available), requests      │
│  projects           id, name, description, created_by           │
│  project_members    id, project_id, user_id                     │
│  project_products   id, project_id, product_id                  │
│  comments           id, project_id, user_id, body               │
│  feedback           id, page, message, created_at               │
│                                                                 │
│  Total: 7 tables    (flat structure, no hierarchy)              │

┌─────────────────────────────────────────────────────────────────┐
│                      AUTH FLOW                                  │
│                                                                 │
│  Admin → /api/auth/invite → sends link → user clicks →         │
│  /invite?token=xxx → creates account → /api/auth/login →       │
│  JWT cookie (jose, HS256, 7-day, httpOnly, secure)              │
│                                                                 │
│  Roles: stored in users.role but NOT enforced per-project       │
│  Membership: project_members (binary: member or not)            │
