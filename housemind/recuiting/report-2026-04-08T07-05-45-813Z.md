# Tech Department Report

## Executive Summary

The Tech department will ship HouseMind MVP in 6-8 weeks with a lean team of 5 specialized agents: Frontend Dev, Backend Dev, Database Engineer, DevOps Engineer, and QA Engineer. Our architecture prioritizes mobile-first delivery using Next.js (Vercel) + FastAPI (Railway) + PostgreSQL, with clear ownership boundaries and contract-first parallel development. Critical path runs through the image carousel → emoji annotation → product linking flow, which we've designed to unblock as early as Week 2.

---

## Key Decisions Made

### Architecture Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Frontend framework | Next.js 14 (App Router) | Best-in-class for mobile performance, Vercel deployment is zero-config |
| Backend framework | FastAPI | Async Python, auto-generated OpenAPI docs, fast development |
| Database | PostgreSQL on Railway | Relational model fits hierarchical projects, Railway simplifies ops |
| Storage | AWS S3 (ap-southeast-1) | Singapore region for Thai users, proven reliability |
| Auth approach | JWT + magic link invites | No passwords to manage, fits invite-only beta model |
| State management | Zustand + React Query | Zustand for UI state, React Query for server cache — avoids Redux complexity |
| Styling | Tailwind CSS | Mobile-first utilities, fast iteration, small bundle |
| i18n | next-intl | Native Next.js integration, handles Thai script well |
| E2E testing | Playwright | Best mobile viewport emulation, runs in CI |

### Team Structure Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Team size | 5 agents + Head of Tech | Minimum viable for our stack; each agent owns a clear domain |
| No Auth Specialist | Backend Dev handles JWT | Invite-only auth is ~3 endpoints, not complex enough for specialist |
| No i18n Specialist | Frontend Dev handles | next-intl is straightforward, Thai/English only |
| Combined scraping role | Backend Dev handles | URL scraping is ~2 endpoints with graceful fallback |
| Dedicated QA | Yes | Mobile-first testing on Samsung A / iPhone SE is non-trivial |

### Scope Decisions

| In MVP | Out of MVP | Revisit When |
|--------|------------|--------------|
| 8 emoji types | Custom emoji | User feedback requests it |
| URL scraping (best-effort) | Automated scraping pipelines | We have supplier partnerships |
| Manual catalog seeding | Catalog CMS | Operations needs to update frequently |
| Basic role permissions | Granular permissions | Enterprise customers need it |
| 2-level project hierarchy | Unlimited nesting | Users hit the limit |

## Risks and Open Questions for PM

### Technical Risks

| Risk | Probability | Impact | Mitigation | Owner |
|------|-------------|--------|------------|-------|
| URL scraping blocked by e-commerce sites | High | Medium | Graceful fallback to manual image URL input; users can paste direct image links | Backend Dev |
| Long-press gesture conflicts with native mobile behavior | Medium | High | Early prototype testing in Week 1; may need custom touch handling | Frontend Dev |
| Thai font rendering issues on older Android | Low | Medium | Test Sarabun + Noto Sans Thai in Week 2; fallback font stack | Frontend Dev |
| Railway cold starts (2-3s on first request) | High | Low | Accept for beta; configure minimum instances if unacceptable ($) | DevOps Engineer |
| S3 image load times in Thailand | Medium | Medium | Using ap-southeast-1 (Singapore); monitor and add CloudFront if needed | DevOps Engineer |
| Invite emails land in spam | Medium | Medium | Use transactional email service (Resend/Postmark); configure SPF/DKIM | DevOps Engineer |

### Open Questions Needing PM Input

| Question | Context | Options | Deadline |
|----------|---------|---------|----------|
| **What happens when scraping fails?** | Many sites block scrapers or have anti-bot measures | A) Show error, require manual URL B) Allow manual image upload C) Both | Week 2 |
| **Maximum image size/count per project?** | Affects storage costs and load times | A) 10 images, 5MB each B) 25 images, 10MB each C) Unlimited (monitor) | Week 1 |
| **Can users delete their own pins?** | Or only architects? Affects permissions model | A) Anyone can delete own B) Only creator + architect C) No deletion | Week 3 |
| **Invite expiration period?** | Security vs. convenience tradeoff | A) 24 hours B) 7 days C) 30 days D) Never | Week 1 |
| **What if a user is invited to multiple projects?** | Need to handle project switching | A) Dropdown switcher B) Separate logins C) Dashboard view | Week 2 |
| **Do we need email notifications for comments?** | Could delay MVP | A) Yes, real-time B) Yes, daily digest C) No, MVP skip | Week 4 |
| **Offline support needed?** | Architects on construction sites may have poor connectivity | A) Yes, PWA with sync B) No, require connection C) Read-only offline | Post-MVP decision |
