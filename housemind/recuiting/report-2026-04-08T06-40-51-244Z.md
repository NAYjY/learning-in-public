# Technology Department Report

## Executive Summary

The Tech team is structured as a 5-person squad with clear ownership boundaries: Frontend, Backend, Database, and Infrastructure, reporting to Head of Technology. Our architecture separates concerns cleanly — Next.js on Vercel handles all UI, FastAPI on Railway handles all business logic, PostgreSQL is the single source of truth, and S3 handles media storage. The 6-8 week MVP timeline is achievable but tight; our biggest risks are the image annotation interaction complexity and web scraping reliability.

---

## Key Decisions Made

### 1. Team Structure: 5 Roles, Clear Ownership

| Role | Primary Ownership | Why Separated |
|------|------------------|---------------|
| Head of Technology | Architecture, API contracts, coordination | Need single decision-maker for cross-cutting concerns |
| Frontend Developer | Next.js UI, interactions, i18n | Image carousel + annotation is substantial specialized work |
| Backend Developer | FastAPI, auth, scraping, business logic | Auth and scraping are security-critical, need focused attention |
| Database Engineer | Schema, queries, migrations, data integrity | Hierarchical projects + permissions require careful modeling |
| Infrastructure Engineer | Vercel, Railway, S3, CI/CD, monitoring | Deploy confidence is critical for beta; needs dedicated focus |

**Rejected alternative:** Separate "Scraping Specialist" role — not enough work for full-time, absorbed into Backend.

### 2. State Management Strategy

- **Server state:** React Query (caching, background refresh, optimistic updates)
- **Client state:** Zustand (UI state, selected pins, active filters)
- **No Redux** — overkill for MVP scope

### 3. Image Annotation Architecture

Pins stored as **percentage coordinates** (x%, y%), not pixels. This ensures annotations remain accurate regardless of:
- Screen size
- Image resize
- Zoom level

```typescript
interface Pin {
  id: string;
  imageId: string;
  x: number;       // 0-100 percentage
  y: number;       // 0-100 percentage
  emoji: EmojiType;
  createdBy: string;
  linkedProductId?: string;
}
```

### 4. Authentication Flow

Magic link / invite-token system:

1. Co-owner enters email in admin panel
2. Backend generates JWT invite token (24h expiry)
3. Email sent with link: `app.housemind.co/invite?token=xxx`
4. User clicks → Backend validates → Sets httpOnly cookie → Redirects to onboarding
5. Subsequent requests use cookie-based JWT

**No passwords in MVP.** Returning users get new magic link.

### 5. Scraping Strategy (MVP)

- **On-demand only** — user pastes URL, we scrape immediately
- **Fallback chain:** OpenGraph → meta tags → first large image
- **Manual override** — user can paste image URL directly if scraping fails
- **No automated pipelines** — out of scope for Phase 1

## API Contract Overview

### Core Endpoints (v1)

```
Authentication:
POST   /auth/invite           — Send invite to email
POST   /auth/verify           — Verify token, set cookie
POST   /auth/refresh          — Refresh JWT
POST   /auth/logout           — Clear cookie

Projects:
GET    /projects              — List user's projects
POST   /projects              — Create project
GET    /projects/:id          — Get project with subprojects
PATCH  /projects/:id          — Update project
DELETE /projects/:id          — Archive project

Images:
GET    /projects/:id/images   — List images in project
POST   /projects/:id/images   — Upload image
DELETE /images/:id            — Remove image

Pins:
GET    /images/:id/pins       — Get pins for image
POST   /images/:id/pins       — Create pin
PATCH  /pins/:id              — Update pin position/emoji
DELETE /pins/:id              — Remove pin

Products:
POST   /products/scrape       — Scrape URL for images
POST   /pins/:id/product      — Link product to pin
GET    /images/:id/products   — Get products for image's pins
GET    /catalog               — Browse curated catalog

Comments:
GET    /images/:id/comments   — Get comments for image
POST   /images/:id/comments   — Add comment
DELETE /comments/:id          — Remove comment

Users:
GET    /users/me              — Current user profile
PATCH  /users/me              — Update profile (language, etc.)
```
