## Executive Summary

After analyzing mobile performance, scraping requirements, hiring market, and AI roadmap, **I recommend Node.js (Next.js) full-stack** for HouseMind MVP. This choice prioritizes faster hiring in Thailand's market, simpler team structure (one language), and adequate performance for our annotation-heavy use case. Python's advantages in scraping and AI are marginal for our Phase 1 scope and can be addressed with microservices later if needed.

---

## Key Decisions Made

### 1. Primary Stack: Next.js 14 (App Router) + TypeScript

**Rationale:**
- **Hiring Reality in Thailand:** 3-5x more senior full-stack TypeScript/Node developers available vs Python+React specialists
- **Salary Data:** ฿80-120K/month for senior Next.js developers in Bangkok (vs ฿100-150K for Python backend + React frontend specialists)
- **Team Simplicity:** One language reduces context switching, PR reviews faster, everyone can work on any feature
- **Performance Trade-off:** Yes, 120KB heavier initial bundle (~0.5s slower on 3G), but acceptable for B2B architect tool where initial load happens once per session

**Technical Architecture:**
```
Frontend: Next.js 14 App Router + React Server Components
Backend: Next.js API Routes + tRPC (type-safe APIs)
Database: PostgreSQL 15 + Prisma ORM
Storage: AWS S3
Auth: next-auth + magic links
Real-time: Pusher (WebSocket alternative, managed)
Deployment: Vercel (frontend) + Railway (background jobs)
```

**Mobile Performance Strategy:**
- Use React Server Components to reduce client JavaScript
- Lazy load annotation canvas (Konva.js) only on project detail pages
- Implement aggressive image optimization (next/image with S3 CDN)
- Progressive Web App (PWA) with service worker for offline annotation drafts
- Target: <3s Time to Interactive on Samsung A34 on 4G

### 2. Annotation System Architecture

**Technology Choice: Konva.js + Optimistic UI**

```typescript
/src/features/annotations
  /components
    AnnotationCanvas.tsx    // Konva wrapper
    CommentThread.tsx       // Role-based comments
  /hooks
    useAnnotations.ts       // Optimistic updates
    useRealtimeSync.ts      // Pusher integration
  /lib
    canvas-serializer.ts    // Save/load annotations
```

**Performance Optimizations:**
- Debounce canvas saves (500ms)
- Virtualize comment threads (react-window)
- Lazy load annotation history
- WebP images with fallback
- Service worker caches annotation state locally

**Target Metrics:**
- 60 FPS on iPhone SE during annotation
- 45+ FPS on Samsung A34
- <100ms annotation save latency
- <2s annotation load time

### 3. Scraping Service (Phase 2, Not MVP)

**Decision: Defer until post-MVP, then Node.js microservice**

**Reasoning:**
- MVP has curated catalog (~100-150 products) manually entered
- Co-owner personally vetting each product = manual URL validation acceptable
- Build scraper only when scaling to 500+ products (Phase 2)

**When needed (Phase 2):**
```
Scraper Service: Node.js + Puppeteer
- Separate Railway/Render deployment
- BullMQ + Redis for job queue
- Scrape on-demand when architect submits product URL
- Fallback: Manual entry form if scrape fails
- Budget: $30/month infrastructure
```

**Phase 1 Alternative (MVP):**
```bash
# Manual curator tool (2 hours development time)
npm run curator:add-product -- --url "https://kohler.com/..."
# Script scrapes, shows preview, curator approves, saves to DB
```

### 4. AI Features Path (Phase 3+)

**Recommendation: Start with API-based approach, language-agnostic**

Both stacks handle API-based AI equally well:

```typescript
// Next.js approach (equally easy as Python)
import { OpenAI } from 'openai'

// Product recommendations
async function getRecommendations(projectId: string) {
  const embedding = await openai.embeddings.create({
    model: "text-embedding-3-small",
    input: projectContext
  })
  
  // Vector search in Postgres with pgvector
  return await db.products.findSimilar(embedding)
}
```

**AI Roadmap (Post-MVP):**
- **Phase 2:** Product recommendations via OpenAI embeddings + pgvector
- **Phase 3:** Image recognition (upload photo → suggest matching products) via OpenAI Vision API
- **Phase 4:** Smart filtering/search via semantic search

**If custom model training needed later:**
- Deploy Python microservice specifically for ML inference
- Main Next.js app calls ML service via internal API
- Keep codebases separated by concern
